// 03 — wiki

Knowledge base

Requirements

Requires zsh as your shell and pacman, which comes standard on all Arch-based distros. The -a flag additionally requires yay for AUR support.

Installing yay

If you don't have yay yet:

git clone https://aur.archlinux.org/yay.git
cd yay && makepkg -si

After that, p -a will work as expected.

How it works

p is a plain zsh function injected into your ~/.zshrc. No binaries, no PATH changes, no daemons. When you type p -i neovim, zsh matches the flag and runs the corresponding pacman command directly.

Passwordless sudo

The installer drops a sudoers drop-in at /etc/sudoers.d/p-package-manager, granting passwordless pacman access for the current user only. It does not touch /etc/sudoers itself, keeping the change isolated and easy to undo.

Supported distros

Any distro using pacman as its package manager: Arch Linux, Manjaro, EndeavourOS, CachyOS, Garuda, Artix, and others. Will not work on Debian, Fedora, or any non-pacman system.

Installing multiple packages

Pass multiple names separated by spaces:

p -i git neovim htop fzf

All arguments after the flag are forwarded directly to pacman, so standard multi-install syntax works as-is.

What -u does

Running p -u executes a full system upgrade via pacman -Syu. Any arguments after -u are ignored — it always upgrades the full system. This is intentional: partial upgrades are unsupported on Arch and can break things.

The .zshrc it installs

The installer replaces your ~/.zshrc with the one from the repo. Your existing file is backed up to ~/.zshrc.bak first. The installed config includes history settings, autocompletion, spell correction, and the p() function.

Updating

Re-run the install script at any time. If it detects an existing clone at ~/.p-package-manager, it runs git pull instead of re-cloning, then refreshes your ~/.zshrc with the latest version.

Uninstalling

Remove the p() function block from ~/.zshrc and delete /etc/sudoers.d/p-package-manager if it exists. Optionally remove ~/.p-package-manager. Nothing else was modified during install.

Unknown flags

Passing an unrecognised flag prints:

Unknown option: X. Try 'p -h' for help.

No command is executed and no packages are touched.

Security notes

The installer requires sudo to write the sudoers drop-in. The NOPASSWD rule is scoped strictly to /usr/bin/pacman for your user only — not a blanket sudo bypass. Check /etc/sudoers.d/p-package-manager after install to verify.

// 04 — contributing

Get involved

01

Fork the repo

Go to GitHub and fork the project to your own account. You'll push your changes there before opening a PR.

02

Clone and edit

git clone https://github.com/YOUR_USERNAME/p-package-manager.git — make your changes to .zshrc or install.sh.

03

Test locally

Source your modified config with source ~/.zshrc and verify everything works. If you changed the installer, run it in a clean environment if possible.

04

Open a pull request

Submit your PR with a clear description of what you changed and why. Keep it focused — one change per PR is ideal.
