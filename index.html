// 03 — wiki

Knowledge base

Requirements

Requires zsh as your shell. Everything else is auto-detected — p checks which package manager is present at runtime and routes commands accordingly. No config files needed.

Supported distros

Arch Linux, Manjaro, EndeavourOS, and other pacman-based distros. Debian, Ubuntu, and derivatives via apt. Fedora and RHEL derivatives via dnf. openSUSE via zypper. macOS via Homebrew (brew). Flatpak works universally on all of the above.

How detection works

p uses command -v to check which package manager binary is on your PATH. It checks in order: pacman → apt → dnf → zypper → brew. First match wins. Flatpak is checked separately and independently of the native manager.

AUR support (Arch)

The -ai, -ar, and -as flags require yay on Arch-based systems. Install it with:

git clone https://aur.archlinux.org/yay.git
cd yay && makepkg -si

Cask support (macOS)

On macOS, -ai installs GUI apps via brew install --cask — stuff like Firefox, VS Code, Spotify. Regular -i still installs CLI tools the normal way. Requires Homebrew to be installed.

Flatpak

Flatpak flags (-fi, -fr, -fs) work on any supported distro as long as flatpak is installed. They are completely independent of the native package manager. Use Flathub IDs like com.spotify.Client.

How it works

p is a plain zsh function injected into your ~/.zshrc. No binaries, no PATH changes, no daemons. When you type p -i neovim, zsh detects your package manager and fires the matching command directly.

Passwordless sudo

On Linux, the installer drops a sudoers drop-in at /etc/sudoers.d/p-package-manager, granting passwordless access for the detected package manager only. It does not touch /etc/sudoers itself. macOS/Homebrew does not need sudo.

Installing multiple packages

Pass multiple names separated by spaces:

p -i git neovim htop fzf

All arguments after the flag are forwarded directly to the underlying package manager.

What -u does

Running p -u executes a full system upgrade. On apt it runs apt update && apt upgrade. On pacman it runs pacman -Syu. Arguments after -u are ignored — it always upgrades everything.

The .zshrc it installs

The installer replaces your ~/.zshrc with the one from the repo. Your existing file is backed up to ~/.zshrc.bak first. The installed config includes history settings, autocompletion, spell correction, and the p() function.

Unknown flags

Passing an unrecognised flag prints:

Unknown option: X. Try 'p -h' for help.

No command is executed and no packages are touched.

Updating

Re-run the install script at any time. If it detects an existing clone at ~/.p-package-manager, it runs git pull instead of re-cloning, then refreshes your ~/.zshrc with the latest version.

Uninstalling

Remove the p() function block from ~/.zshrc and delete /etc/sudoers.d/p-package-manager if it exists. Optionally remove ~/.p-package-manager. Nothing else was modified during install.

Security notes

The installer requires sudo to write the sudoers drop-in. The NOPASSWD rule is scoped strictly to the detected package manager binary for your user only — not a blanket sudo bypass. Check the drop-in after install to verify.

// 04 — contributing

Get involved

01

Fork the repo

Go to GitHub and fork the project to your own account. You'll push your changes there before opening a PR.

02

Clone and edit

git clone https://github.com/YOUR_USERNAME/p-package-manager.git — make your changes to .zshrc or install.sh.

03

Test locally

Source your modified config with source ~/.zshrc and verify everything works. If you changed the installer, run it in a clean environment if possible.

04

Open a pull request

Submit your PR with a clear description of what you changed and why. Keep it focused — one change per PR is ideal.
